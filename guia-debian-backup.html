Guia de Configuração Essencial para Servidores Debian

Introdução

Neste tutorial, abordaremos os principais ajustes necessários para otimizar um servidor ou VPS com Debian. O Debian é reconhecido por sua estabilidade e desempenho, mas requer algumas configurações adicionais para melhor integração com ferramentas de monitoramento e administração.

Pré-requisitos

  • Sistema Debian Linux instalado
  • Versão do Debian: 12 (Bookworm) ou superior

Download do Debian

Para obter a imagem ISO mais recente:

  • Site oficial: https://www.debian.org
  • Link direto: https://cdimage.debian.org/debian-cd/current/amd64/iso-cd/
  • Arquivo recomendado: debian-V.S.R-amd64-netinst.iso(onde V=versão, S=sub-versão, R=release)
  • Para versões em desenvolvimento: https://cdimage.debian.org/cdimage/weekly-builds/amd64/iso-cd/

1. Configuração de Nome do Servidor e DNS

É fundamental que todo servidor tenha seu nome corretamente configurado e sincronizado com o endereço IP e registros DNS reversos.

Exemplo de configuração:

Nome do servidor: srv101.exemplo.com.br

IPv4: 45.255.128.2

IPv6: 2804:cafe:beba:c0ca::2

Configuração de DNS:

Bash 

# Definindo nome do servidor:
hostnamectl set-hostname srv101.exemplo.com.br

# Conferindo:
hostname -f
cat /proc/sys/kernel/hostname

Configuração de arquivos hosts e resolv.conf:

bash 

# Configurar /etc/hosts
(
  echo '127.0.0.1   localhost'
  echo '::1         localhost'
  echo'ff02::1     ip6-allnodes'
  echo 'ff02::2     ip6-allrouters'
  echo '45.255.128.2        srv101.exemplo.com.br'
  echo '2804:cafe:beba:c0ca::2   srv101.exemplo.com.br'
) > /etc/hosts

# Configurar /etc/resolv.conf
(
  echo 'nameserver 8.8.8.8'
  echo 'nameserver 1.1.1.1'
) > /etc/resolv.conf
